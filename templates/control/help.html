Segmentation - Help

  • Click the "segment" button to enter segmentation mode.
  • Your document will be displayed as a running text with green lines separating any elementary discourse units (EDU) that have already been segmented
  • To add more segments, click between words to create separators
  • To merge adjacent segments, click on the separator between them
  • You can undo / redo your actions using the appropriate buttons
  • Don't forget to save your work before returning to edit mode
  • If a guideline link has been set for the current project, the guidelines button will allow you to access them.

Edit structure - Help

In edit mode, each elementary discourse unit (EDU) has its text under a line with a number giving its position in the text. Similarly, groups of units have a range of numbers. You can connect nodes, unlink them, and group them together under spans and multinuclear nodes. You can undo and redo actions at any point using the appropriate buttons. Don't forget to save your work!

If a guideline link has been set for the current project, the guidelines button will allow you to access guidelines for your project.

Connections

  • To connect nodes, drag from the numbers under one unit to the numbers under a target unit
  • To change the relation between two units, use the drop down list on the connection between them
  • To unlink a node from the graph, click its button - all of the nodes connected above it will be unlinked

Spans

  • Use the button to add a span above a node. The span will group together all of the nodes connected to that node

Multinuclear relations

  • Use the button to create a multinuclear node (multinuc) above a node. The multinuc can have multiple child nodes with the same relation (e.g. a set of items in contrast to each other)
  • To change the relation of a multinuc to its children, change the relation box of any of its children
  • When connecting a new node to a multinuc node, it will be added as a multinuc child by default. You can change the relation to a satellite relation by choosing "change to satellite" in the relation box

Signal annotations

  • To use discourse relation signal annotations, make sure signaling annotation is switched on in the Administration > Database tab
  • You can use the button next to the relation chooser to open the signal annotation tray and create a new signal annotation
  • Select a signal type and subtype, then click on any words that should be included in the signal annotation
  • You can click on signal annotations in the tray to see associated words, or use the Show all signals button
  • Click Save Changes when done to close the tray

Open - Help

  • To open a file for editing, select it from the list and click open
  • If no files have been imported yet, an administrator must import them from the administrator interface
  • If a file has been imported but you cannot see it in the open file area, your user may not have been assigned the right to edit the file. An administrator can assign the file to you from the administrator interface

Administration - Help

The administration area has five tabs:

Project management

  • All documents belong to a project - you must create at least one project to import into
  • Deleting a project will also delete all documents within that project
  • You can set a URL giving guidelines for the selected annotation project(s)

User management

  • User management is only available on server installations (not in local mode)
  • You can add or delete users (user names may not contain spaces or special characters except underscores and cannot be: 'config', 'local', 'default', 'temp', 'emails', '_orig' or 'pending'
  • You can assign/unassign a user to a document. This will create/delete a copy of the document for that user to annotate

Document management

  • Deleting a document will completely remove it from the database - to delete only a single user's annotated version, unassign the document in User Management
  • Exporting documents will result in one *.rs3 file for each annotated version of each exported document

Import

  • You can import documents from RSTTool files (*.rs3) or plain text files (one discourse unit per line)
  • Documents must be imported into a project, and you can also create new projects. Project names should not contain spaces or special characters except the underscore.
  • For plain text files use the file default_relations.tab to edit the relations that will be available after import
  • Exporting documents will result in one *.rs3 file for each annotated version of each exported document

Database

  • Here you can turn detailed logging on/off (annotation steps will be saved in the table logging)
  • You can disable/enable the span and multinuc buttons (for non-RST annotation, e.g. binary discourse relations)
  • You can also update the database to the latest schema (backup first just in case) or reset the database to its initial state.
  • Resetting will delete all annotation projects. Only use this option if the database becomes corrupt or you wish to delete all data.
